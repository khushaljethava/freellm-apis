# Competitor Analysis — freellm.site

_Note: freellm.site is DR 0 (new). Competitors below are the incumbents ranking for the target cluster. Validate exact metrics with Ahrefs/Semrush before acting._

## Competitive landscape

| Competitor | Type | Strength | Weakness we exploit |
|------------|------|----------|--------------------|
| github.com/cheahjs/free-llm-api-resources | GitHub repo (list) | High DR, dev-trusted, top-ranking for the term | Not a website — no schema, poor UX, no per-provider pages. We can out-UX and out-rank on long tail. |
| Individual provider docs (Groq, Gemini…) | First-party | Authoritative for own brand | Only cover themselves — we win comparison + "which free API" intent. |
| Dev blogs / Medium / dev.to listicles | Editorial | Some authority, fresh | Go stale fast; no live data. Our freshness + completeness wins. |
| Reddit / HN threads | Community | Rank for long-tail Q's, high trust | Unstructured; we provide the structured citable answer. |

## Where we win (gap map)
1. **Completeness** — 105 providers vs typical 20–30. Own the long tail of provider-name queries.
2. **Structured per-provider pages** — the GitHub repo (our top competitor) has none. This is our schema + UX moat.
3. **Freshness** — rate limits change monthly; open-source data + automation keeps us current. Competitors decay.
4. **Comparison intent** — "X vs Y free tier" is under-served by first-party docs and stale listicles.
5. **GEO/AI citation** — clean data tables + answer-first prose make us the easy citation for AI Overviews.

## Where we're behind (must close)
1. **Domain authority** — DR 0 vs established repos/blogs. → Pillar 3 link acquisition is existential.
2. **Content depth on provider pages** — thinner than first-party docs. → Phase 2 uniqueness work.
3. **Brand recognition** — none yet. → GitHub funnel + community launch + quarterly report.

## Keyword gap priorities (validate volumes)
- Head: `free llm api`, `free llm api key`, `free ai api`, `openai compatible api`
- Modifier long-tail: `[provider] free tier`, `[provider] rate limits`, `[provider] api key free` × 105 providers
- Comparison: `groq vs gemini`, `[a] vs [b] free tier`
- Use-case: `free llm api for chatbot / rag / coding / discord bot`

## Strategic takeaway
Our single biggest competitor is a **GitHub list with high DR but zero web-page structure**. Beating it means (a) earning links to close the DR gap, and (b) exploiting everything a repo can't do: per-provider indexable pages, schema, comparisons, and freshness. Both are already in the roadmap (Pillars 1 & 3).
